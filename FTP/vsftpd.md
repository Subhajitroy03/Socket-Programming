To understand these four commands, it helps to use a very simple mental model.

Think of your local computer (the machine you are sitting at) and the remote FTP server as two different rooms. FTP commands are just instructions telling the server which way to move the boxes (files).

get / mget = Receive (Download from Server → Local Computer)

put / mput = Send (Upload from Local Computer → Server)

The letter "m" simply stands for Multiple.

1. Receiving Files (Downloading)
get (Get ONE file)
Logic: You know the exact name of a single file on the server, and you want to pull it onto your computer.

Syntax: get filename.txt

mget (Multiple Get)
Logic: You want to download more than one file at the same time, often using a wildcard asterisk (*) to match a pattern.

Syntax: mget *.txt (This grabs all text files).

Student Trap: By default, mget will ask you Y/N? for every single file before downloading it. To turn this annoying feature off before running the command, type prompt and hit Enter.

2. Sending Files (Uploading)
put (Put ONE file)
Logic: You have a single file on your local machine, and you want to push a copy of it up to the server.

Syntax: put localfile.txt

mput (Multiple Put)
Logic: You have a batch of files on your local computer and want to upload them all in one go.

Syntax: mput image1.png image2.png or mput *.png

Quick Reference Summary Table
Command	Action	Direction	How many files?
get	Receive	Server → Local	Exactly one
mget	Receive	Server → Local	Multiple (uses wildcards)
put	Send	Local → Server	Exactly one
mput	Send	Local → Server	Multiple (uses wildcards)

The "Where did it go?" Mistake: Beginners often assume files are downloaded to or uploaded from the directory where vsftpd is installed. This is incorrect. * get downloads files into the local directory you were standing in on your PC right before you typed the ftp command.

If you want to change your local directory while remaining inside the FTP session, use lcd (Local Change Directory), for example: lcd /home/student/Downloads.

Binary vs. ASCII: By default, some older FTP clients transfer files in ASCII (text) mode. If you try to get or put an image, zip file, or executable using ASCII, the file will get corrupted. Get into the habit of typing binary and pressing Enter as soon as you log into an FTP server.

To list all the commands associated with vsftpd, we need to split them into two different categories, because "vsftpd commands" usually means one of two things to a student:

System Administration Commands: The Linux commands you run in your terminal to manage, start, or configure the vsftpd server application itself.

Interactive FTP Client Commands: The commands you type inside the FTP prompt once you connect to the server (like get and put).

Here is the complete breakdown for both.

1. System Administration Commands (Managing the Server)
These commands are executed directly in your Linux terminal (usually requiring sudo) to control the vsftpd background service.

Command	Use / Description
sudo systemctl start vsftpd	Starts the vsftpd server service if it is stopped.
sudo systemctl stop vsftpd	Stops the vsftpd server service immediately.
sudo systemctl restart vsftpd	Stops and then immediately starts the service. Crucial for applying changes made to /etc/vsftpd.conf.
sudo systemctl status vsftpd	Displays whether the server is running, stopped, or experiencing errors.
sudo systemctl enable vsftpd	Configures vsftpd to start automatically whenever the Linux computer boots up.
sudo systemctl disable vsftpd	Prevents vsftpd from starting automatically at boot.
vsftpd -v	Displays the current version number of vsftpd installed on your system.
2. Interactive FTP Client Commands (Managing Files)
Once you type ftp localhost or ftp <IP_Address> and successfully log in, your terminal prompt changes to ftp>. These are the most common commands used to navigate and transfer files.

File Transfer Commands
get <remote-file>: Downloads a single file from the server to your local machine.

mget <pattern>: Downloads multiple files from the server (e.g., mget *.jpg).

put <local-file>: Uploads a single file from your local machine to the server.

mput <pattern>: Uploads multiple files to the server.

append <local-file> <remote-file>: Appends the contents of a local file to the end of a file on the FTP server.

Navigation & Directory Commands
ls: Lists the files and folders in the current directory on the server.

dir: Similar to ls, provides a detailed list of files on the server.

cd <directory>: Changes the current working directory on the server.

cdup: Moves up one level to the parent directory on the server (same as cd ..).

pwd: "Print Working Directory" — shows you exactly which folder you are currently standing in on the server.

lcd <directory>: "Local Change Directory" — changes the folder you are standing in on your local computer without disconnecting you from the FTP server.

File Management Commands
mkdir <directory-name>: Creates a new folder on the server.

rmdir <directory-name>: Deletes an empty folder on the server.

delete <file-name>: Deletes a single file on the server.

mdelete <pattern>: Deletes multiple files on the server.

rename <old-name> <new-name>: Renames a file or folder on the server.

Session & Environment Commands
binary: Switches the transfer mode to Binary. Use this for images, zip files, and executables to prevent corruption.

ascii: Switches the transfer mode to text (ASCII). Used strictly for plain text files.

prompt: Toggles interactive prompting on or off. Turning it off stops mget/mput/mdelete from asking you "Are you sure? (y/n)" for every individual file.

help or ?: Lists all available interactive commands supported by your client.

help <command>: Explains what a specific command does (e.g., help get).

bye, quit, or exit: Safely terminates the FTP session and returns you to your normal Linux terminal.

Your ability to run commands like put, mkdir, or delete is strictly dictated by the rules set in /etc/vsftpd.conf (like write_enable=YES) and the underlying Linux file permissions of the user's home directory. If Linux permissions say "Read-Only", the FTP client will throw a 550 Permission denied error, even if vsftpd is configured correctly.

# VSFTPD (Very Secure FTP Daemon) Beginner Study Guide

A comprehensive, lab-tested reference manual for installing, configuring, managing, and validating an operational FTP Server using `vsftpd` over a local loopback interface.

---

## 1. Installations

To set up an operational environment on Ubuntu, Debian, or WSL2, run the following commands sequentially:

* **`sudo apt update`** *Refreshes package repositories. This is a crucial first step to ensure your package manager fetches the most up-to-date dependencies.*
* **`sudo apt install vsftpd`** *Installs the actual FTP server daemon software that runs in the background.*
* **`sudo apt install net-tools`** *(Optional) Installs legacy utilities like `ifconfig` to quickly view your internal IP architecture.*
* **`sudo apt install inetutils-ftp -y`** *Installs the interactive command-line FTP client tool used to establish outbound connections to servers.*

---

## 2. Basic Server Management Commands

These system utility commands control the lifecycle of the background service daemon via `systemctl`:

* **`sudo systemctl start vsftpd`** Starts the background service daemon thread.
* **`sudo systemctl status vsftpd`** Displays the active state of the service. Look for a bright green `active (running)` string to verify health.
* **`sudo systemctl stop vsftpd`** Shuts down the FTP server engine immediately.
* **`sudo systemctl restart vsftpd`** Stops and immediately spins up the daemon process. **Mandatory** after making any structural modifications to `/etc/vsftpd.conf`.
* **`sudo systemctl enable vsftpd`** Configures the system init layer to start the FTP service automatically whenever the Linux machine boots up.
* **`sudo systemctl disable vsftpd`** Prevents the FTP server from launching automatically on system startup.
* **`vsftpd -v`** Prints out the current version number of the `vsftpd` binary compiled on your machine.

---

## 3. Client Connection Step

To log in and navigate your server space internally, run:

1. **`ftp <IP Address>`** *Initiates a standard network connection. Use `127.0.0.1` (local loopback) for testing purposes.*
